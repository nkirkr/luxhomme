# SEO и GEO: полная инструкция

Инструкция по поисковой оптимизации (SEO) и оптимизации для генеративных поисковых систем (GEO) для сайтов на базе Bazasite.

---

## 1. GEO — Generative Engine Optimization

GEO — оптимизация контента для AI-поисковых систем: Google AI Overviews, Perplexity, Yandex Neuro, ChatGPT Search. В 2025-2026 до 30% информационных запросов закрываются нейроответами без клика. Страницы, цитируемые в AI-ответах, получают на 35% больше органических кликов.

### 1.1 Принципы GEO-контента

| Принцип               | Действие                                                        |
| --------------------- | --------------------------------------------------------------- |
| Ответ первым          | Первые 40-80 слов — прямой ответ на вопрос, затем раскрытие     |
| Заголовки-вопросы     | H2/H3 формулировать как вопросы пользователя                    |
| Фактическая плотность | Конкретные числа, статистика, даты, именованные сущности        |
| Семантическая полнота | Покрыть все подтемы: определение, сравнение, шаги, ограничения  |
| Атрибуция автора      | Имя, должность, компетенции (E-E-A-T)                           |
| Нейтральный тон       | Избегать рекламных формулировок ("только у нас", "лидер рынка") |
| Schema.org            | Структурированная разметка увеличивает цитирование до 28%       |
| Свежесть              | Обновлять даты, статистику, факты минимум раз в месяц           |

### 1.2 Структура GEO-оптимизированной страницы

```
H1: Точный ответ на запрос (50-60 символов)

[Первый абзац: прямой ответ на вопрос — 40-80 слов]

H2: Как это работает? / Что выбрать?
[Детальное раскрытие с числами и фактами]

H2: Сравнение / Плюсы и минусы
[Таблица или список с конкретными данными]

H2: Пошаговая инструкция
[Нумерованный список действий]

H2: Часто задаваемые вопросы
[FAQ-блок с разметкой FAQPage]
```

### 1.3 AI-краулеры и robots.txt

В проекте настроены правила для AI-краулеров (`src/app/robots.ts`):

| User-Agent        | Правило  | Причина                                       |
| ----------------- | -------- | --------------------------------------------- |
| `ChatGPT-User`    | Allow    | Даёт обратные ссылки при цитировании          |
| `PerplexityBot`   | Allow    | Трафик конвертируется в 10-14% (vs 2% Google) |
| `GPTBot`          | Allow    | С 2025 даёт атрибуцию и обратные ссылки       |
| `ClaudeBot`       | Allow    | Цитирование с ссылками на источник            |
| `Google-Extended` | Disallow | Только обучение, без атрибуции                |

### 1.4 GEO для Yandex Neuro

Yandex Neuro обработал более 2.9 млрд запросов. Особенности:

- Анализирует смысл, а не ключевые слова
- Игнорирует анонимный контент — нужны имена авторов с квалификацией
- Рекламный язык снижает видимость
- Фокус на конкретных утверждениях и фактах

## 2. Техническое SEO для Next.js

### 2.1 Рендеринг

| Стратегия         | SEO | Когда использовать                           |
| ----------------- | --- | -------------------------------------------- |
| SSG (Static)      | 5/5 | Маркетинговые страницы, документация, блог   |
| ISR (Incremental) | 5/5 | Контент, требующий периодического обновления |
| SSR (Server)      | 4/5 | Динамический персонализированный контент     |
| CSR (Client)      | 2/5 | Никогда для SEO-критичного контента          |

**Правило:** если контент важен для SEO, рендерить на сервере. `"use client"` — только для интерактивности.

### 2.2 Метаданные

Каждая страница должна иметь уникальные метаданные через `generateMetadata()`:

```typescript
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const product = await getProduct(params.slug)
  return {
    title: `${product.name} — купить ${product.category}`,
    description: `${product.shortDesc}. Цена от ${product.price}₽. Доставка.`,
    alternates: {
      canonical: absoluteUrl(`/products/${params.slug}`),
    },
    openGraph: {
      title: product.name,
      description: product.shortDesc,
      images: [product.mainImage],
      type: 'website',
    },
  }
}
```

Требования:

- Title: 50-60 символов, основное ключевое слово в начале
- Description: 150-160 символов, написано для CTR
- Canonical URL на каждой странице
- Open Graph теги на всех индексируемых страницах

### 2.3 Core Web Vitals

| Метрика     | Цель                                                                   | Как достичь в Next.js |
| ----------- | ---------------------------------------------------------------------- | --------------------- |
| LCP < 2.5s  | `next/image` с `priority`, `next/font` с `display: swap`               |
| CLS < 0.1   | Всегда указывать width/height для изображений, skeleton-загрузки       |
| INP < 200ms | Server Components по умолчанию, `next/dynamic` для тяжёлых компонентов |

### 2.4 Crawl Budget

- Flat-архитектура: важные страницы в 3 клика от главной
- `robots.ts`: закрывать `/api/`, `/dashboard/`, `/admin/`, `/checkout/`
- `sitemap.ts`: все индексируемые страницы с `lastModified`
- SSG/ISR: снижает нагрузку на краулеры
- Canonical теги: предотвращают индексацию дублей

### 2.5 Чеклист технического SEO

- [ ] `generateMetadata()` на каждой странице с уникальными title/description
- [ ] `alternates.canonical` на каждой странице
- [ ] `sitemap.ts` покрывает все индексируемые URL
- [ ] `robots.ts` блокирует служебные маршруты
- [ ] JSON-LD на каждой странице (тип зависит от контента)
- [ ] Breadcrumbs с разметкой BreadcrumbList
- [ ] OG-изображения для социальных сетей
- [ ] `next/image` для всех изображений (WebP/AVIF)
- [ ] `next/font` для шрифтов (no layout shift)
- [ ] Скрипты аналитики через `next/script` с `strategy="afterInteractive"`

## 3. E-commerce SEO

### 3.1 Фасетное SEO

Фасетная навигация (фильтры) — ключевой инструмент для e-commerce SEO. Каждая значимая комбинация фильтров должна стать полноценной SEO-страницей с ЧПУ-URL.

**Стратегия индексации фасетов:**

| Тип фасета                       | Стратегия                            | Пример URL              |
| -------------------------------- | ------------------------------------ | ----------------------- |
| Высокий спрос (категория, бренд) | Индексировать, ЧПУ-URL               | `/products/nike/`       |
| Средний спрос (материал, стиль)  | Индексировать, canonical на родителя | `/products/leather/`    |
| Низкий спрос (цвет, размер)      | `noindex`, canonical на родителя     | `/products?color=red`   |
| Комбинации 3+ фильтров           | Блокировать в robots.txt             | `/products?a=1&b=2&c=3` |

**Правила:**

- Один смысл = один URL (не допускать `/kovrolin/4-metra/` и `/kovrolin/4m/`)
- Индексировать комбинации до 2-3 атрибутов
- Минимум 3-5 товаров в выборке для создания отдельной страницы
- GET-параметры — через `Clean-param` в robots.txt

### 3.2 Динамическая генерация метатегов

Для фасетных и категорийных страниц метатеги генерируются по формуле из словарей атрибутов:

```typescript
import { generateCategoryMeta } from '@/lib/seo'

export async function generateMetadata({ params, searchParams }) {
  return generateCategoryMeta({
    category: 'Ковролин',
    scenario: 'для офиса',
    filters: {
      class: '33 класс',
      color: 'серый',
      width: '4 м',
    },
    siteName: 'Kovrolin.ru',
  })
}
// Результат:
// title: "Ковролин для офиса 33 класс, серый, 4 м — купить, цены | Kovrolin.ru"
// h1: "Ковролин для офиса 33 класс, серый, 4 м шириной"
```

### 3.3 Сценарные страницы

Создавать отдельные страницы по сценариям использования:

```
/products/                        → Все товары
/products/for-home/               → Для дома
/products/for-home/bedroom/       → Для спальни
/products/for-home/kids-room/     → Для детской
/products/for-office/             → Для офиса
/products/for-hotel/              → Для гостиниц
```

Каждая сценарная страница имеет:

- Уникальный H1, title, description
- Предустановленные фильтры
- SEO-текст с перелинковкой
- Релевантные статьи из блога

### 3.4 Карточки товаров

Обязательные элементы:

- Характеристики в структурированном виде (таблица)
- Блок "Подходит для..." (сценарии использования)
- FAQ по типичным вопросам (с разметкой FAQPage)
- Похожие товары / Другие цвета коллекции
- Сопутствующие товары (cross-sell)
- Сценарные фото (товар в интерьере)

JSON-LD для товара:

```typescript
<ProductJsonLd
  name="Ковролин AW Marcus 39"
  description="Бытовой ковролин для спальни и гостиной"
  image="/products/aw-marcus-39.jpg"
  url={absoluteUrl('/products/aw-marcus-39')}
  price={2100}
  currency="RUB"
  availability="InStock"
  brand="Associated Weavers"
  sku="AW-MARCUS-39"
  ratingValue={4.5}
  reviewCount={12}
/>
```

### 3.5 Внутренняя перелинковка

Каждое значимое свойство в карточке товара — ссылка в соответствующий раздел:

- Бренд → страница бренда
- Категория → раздел каталога
- Класс → фасетная страница
- Цвет → фильтр по цвету

На категорийных страницах:

- Блок "Полезные статьи" → релевантные гайды
- Тегоблако → ссылки на фасетные страницы
- Блок "Похожие категории" → смежные разделы

### 3.6 Мегаменю для SEO

Мегаменю создаёт внутренний ссылочный индекс сайта. Структура:

```
Каталог
├── По назначению
│   ├── Для дома
│   ├── Для офиса
│   └── Для гостиниц
├── По параметрам
│   ├── Высокий ворс
│   ├── 33 класс
│   └── 4 м шириной
└── По брендам
    ├── Tarkett
    ├── ITC
    └── Zarteks
```

## 5. SEO для Yandex

### 5.1 Факторы ранжирования Yandex (2025-2026)

Yandex радикально сместился к **поведенческим метрикам**:

| Фактор           | Вес     | Описание                                               |
| ---------------- | ------- | ------------------------------------------------------ |
| Last Click       | Высший  | Пользователь закрыл поиск после визита = задача решена |
| Dwell Time       | Высокий | Время на сайте; короткие визиты = плохой контент       |
| CTR в SERP       | Высокий | Высокий CTR vs конкуренты = преимущество               |
| Повторные визиты | Средний | Возвращающиеся пользователи = ценность                 |
| Глубина скролла  | Средний | Насколько глубоко скроллят контент                     |

**Что больше НЕ работает:**

- Закупка ссылок — Yandex детектирует 99.9% покупных ссылок
- Переспам ключевыми — нейросети оценивают смысл, а не плотность

### 5.2 ИКС (Индекс Качества Сайта)

ИКС складывается из:

- Размер и вовлечённость аудитории
- Поведенческие метрики (время на странице, bounce rate, глубина сессии)
- Доверие пользователей
- Качество внешних ссылок (не количество)
- Техническая стабильность (Core Web Vitals)
- Брендовые запросы
- Присутствие в сервисах Yandex (Карты, Бизнес, Дзен)

**Как улучшать ИКС:**

1. Качественный контент — калькуляторы, инфографика, интерактив
2. Внутренняя перелинковка между связанными страницами
3. Core Web Vitals оптимизация
4. Наращивание аудитории с повторными визитами
5. Брендовые запросы (узнаваемость)
6. Отзывы (Yandex Карты, отзовик, irecommend)
7. Регистрация в Yandex Бизнес
8. Публикации на Yandex Дзен

### 5.3 Yandex-специфичные настройки

В `robots.ts` проекта добавлена директива `Host` для Yandex. Дополнительно:

- Зарегистрировать сайт в **Yandex Вебмастер** (отдельно от Google Search Console)
- Настроить **региональную привязку** в Вебмастере
- `Clean-param` для GET-параметров фильтров (предотвращает дубли)
- **Turbo-страницы** для мобильной скорости (опционально)
- **Yandex Метрика** для поведенческих сигналов (уже в проекте, feature flag `NEXT_PUBLIC_FEATURE_ANALYTICS`)

## 6. Контент-стратегия и SEO

### 6.1 Связка каталога и блога

Контент не должен жить отдельно от каталога. Схема связей:

```
Категория "Для офиса"
  → блок "Полезные статьи"
      → "Как выбрать покрытие для офиса"
      → "Что важно знать при выборе"

Статья "Как выбрать покрытие для офиса"
  → ссылки на:
      /products/for-office/
      фасеты по классу, цвету
      конкретные рекомендованные товары

Карточка товара
  → блок "Читайте также"
      → статьи по выбору и уходу
```

### 6.2 Типы контента для SEO

| Тип             | Примеры               | Целевые запросы               |
| --------------- | --------------------- | ----------------------------- |
| Гайды по выбору | "Как выбрать X для Y" | информационные                |
| Сравнения       | "X vs Y: что лучше"   | коммерческие + информационные |
| Инструкции      | "Как установить X"    | информационные (DIY)          |
| Подборки/топы   | "Топ-10 X для Y"      | коммерческие                  |
| FAQ             | "Можно ли X на Y?"    | информационные                |

### 6.3 Кластеризация семантики

Семантическое ядро строится по матрице:

```
назначение × параметры × дизайн × гео × коммерческий маркер
```

Каждый кластер маппится на тип страницы:

| Кластер         | Тип            | Пример запроса              | Посадочная        |
| --------------- | -------------- | --------------------------- | ----------------- |
| Категорийный    | Коммерческий   | "X для офиса купить"        | Раздел каталога   |
| Параметрический | Коммерческий   | "X 33 класс серый"          | Фасетная страница |
| Брендовый       | Коммерческий   | "X Tarkett Dragon"          | Страница бренда   |
| Информационный  | Информационный | "как выбрать X для детской" | Статья в блоге    |
| Локальный       | Транзакционный | "купить X москва"           | Категория + гео   |

## 7. SEO-утилиты в проекте

### 7.1 Хелперы (`src/lib/seo.ts`)

```typescript
import { generateCanonicalUrl, generateAlternateUrls, generateProductMeta } from '@/lib/seo'

// Canonical URL
const canonical = generateCanonicalUrl('/products/widget')
// → "https://example.com/products/widget"

// Hreflang для i18n
const alternates = generateAlternateUrls('/products/widget', ['en', 'ru'])
// → { en: "https://example.com/en/products/widget", ru: "https://example.com/ru/products/widget" }

// Метаданные для товара
const meta = generateProductMeta({
  name: 'Widget Pro',
  description: 'Best widget ever',
  price: 2999,
  currency: 'RUB',
  image: '/products/widget.jpg',
  slug: 'widget-pro',
  siteName: 'My Shop',
})
```

### 7.2 Sitemap (`src/app/sitemap.ts`)

Автоматически генерирует карту сайта из:

- Статические маршруты: `/`, `/about`, `/contact`, `/search`
- Блог: `/blog` + `/blog/[slug]` (из CMS)
- Магазин: `/products` + `/products/[slug]` (из Shop-адаптера)

### 7.3 Robots.txt (`src/app/robots.ts`)

- Разрешает основным краулерам и AI-ботам
- Блокирует `/api/`, `/dashboard/`, `/admin/`, `/checkout/`
- Включает `Host` для Yandex
- Ссылается на sitemap.xml
