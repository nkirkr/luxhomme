# Оркестрация конвейера just-do-it

## Структура конвейера

Агенты работают строго последовательно. Каждый читает входной файл самостоятельно:

```
Пользователь → @jdi_analyst  → analyst_output.md
                                    ↓
                @jdi_architect → architect_output.md
                                    ↓
                @jdi_engineer  → engineer_output.md
                                    ↓
                @jdi_developer → developer_output.md
                                    ↓
                @jdi_judge     → judge_output.md      ← опционально
                                    ↓
                @jdi_writer    → writer_output.md     ← опционально
```

## Главное правило — не передавать контекст вручную

Не собирай контекст и не пересылай результаты между агентами. Каждый агент сам читает нужный файл из `ai/tasks/task-<NN>-<название>/`.

Твоя роль — вызвать агента и указать папку задачи.

**Исключение:** аналитику передаётся задача напрямую — его входные данные это слова пользователя, а не файл.

## Что передавать каждому агенту

| Агент            | Что передавать                                     |
| ---------------- | -------------------------------------------------- |
| `@jdi_analyst`   | Задачу пользователя дословно + путь к папке задачи |
| `@jdi_architect` | Только путь к папке задачи                         |
| `@jdi_engineer`  | Только путь к папке задачи                         |
| `@jdi_developer` | Только путь к папке задачи                         |
| `@jdi_judge`     | Только путь к папке задачи                         |
| `@jdi_writer`    | Только путь к папке задачи                         |

Не читай содержимое `.md` файлов. Не пересказывай их агентам. Агент сам знает что читать — это в его системном промпте.

## Алгоритм каждого шага

1. Пользователь называет агента и задачу (или путь)
2. Вызови агента, передав только то что указано в таблице
3. Дождись завершения — агент пишет свой `.md` файл
4. Сообщи результат по формату ниже
5. **Остановись. Жди следующей команды.**

## Что считается командой продолжить

✅ "запусти архитектора", "теперь инженер", "следующий", "продолжай"

❌ Молчание — не команда  
❌ Логическая очевидность следующего шага — не команда  
❌ Завершение предыдущего агента — не команда

## Исключение: полный прогон

Если пользователь явно говорит "запусти весь конвейер" или "прогони всех подряд":

- Выполняй шаги последовательно
- Продолжай следующий только если предыдущий завершился успешно (файл создан)
- Контекст между агентами не передаёшь — каждый читает файл сам

## Формат сообщения после каждого агента

```
✅ @jdi_analyst завершил работу
📄 ai/tasks/task-<NN>-<название>/analyst_output.md
⏸ Жду следующей команды.
```
